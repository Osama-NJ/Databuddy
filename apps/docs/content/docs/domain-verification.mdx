# üîí Domain Verification

Domain verification ensures that only authorized domains can send analytics data to your Databuddy project. This prevents unauthorized usage and protects your analytics data from spam or malicious traffic.

<Callout type="warn">
**Required First Step**: You must verify your domain before you can create projects and track analytics data.
</Callout>

## üõ°Ô∏è Why Domain Verification?

<Cards>
  <Card title="Security Benefits" icon="üîí">
    - **Prevent Unauthorized Usage** - Only verified domains can send data
    - **Protect Against Spam** - Block fake traffic from unverified sources
    - **Data Integrity** - Ensure analytics come from legitimate sources
    - **Cost Control** - Prevent unexpected usage charges
  </Card>
  
  <Card title="How It Works" icon="‚öôÔ∏è">
    1. Add a domain to your Databuddy project
    2. Databuddy generates a unique verification token
    3. Add a DNS TXT record with the token
    4. Databuddy verifies the DNS record
    5. Domain is authorized for analytics tracking
  </Card>
</Cards>

## üöÄ Domain Verification Process

<Steps>
<Step>

## Access Domain Management

1. Go to your [Databuddy Dashboard](https://app.databuddy.cc)
2. Navigate to **Settings** ‚Üí **Domains**
3. Click **Add Domain**

<Callout type="info">
If you don't have an account yet, [sign up for free](https://app.databuddy.cc) first.
</Callout>

</Step>
<Step>

## Enter Your Domain

Enter your domain name **without** the protocol or paths:

<Tabs items={['Correct Format', 'Common Mistakes']}>
<Tab value="Correct Format">

```
‚úÖ Correct examples:
example.com
www.example.com
app.example.com
blog.example.com
subdomain.example.com
```

</Tab>
<Tab value="Common Mistakes">

```
‚ùå Avoid these formats:
https://example.com     (don't include protocol)
http://example.com      (don't include protocol)  
example.com/path        (don't include paths)
example.com:8080        (don't include ports)
www.example.com/        (don't include trailing slash)
```

</Tab>
</Tabs>

</Step>
<Step>

## Get DNS Instructions

After adding your domain, you'll receive:
- **DNS Record Name**: `_databuddy.yourdomain.com`
- **DNS Record Value**: `databuddy_abc123...` (unique verification token)
- **TTL**: 300 seconds (recommended)

<Callout type="warn">
**Important**: Keep this token secure and don't share it publicly. Generate a new one if it's compromised.
</Callout>

</Step>
<Step>

## Configure DNS Record

Add a TXT record to your domain's DNS settings:

<Tabs items={['Record Details', 'DNS Format']}>
<Tab value="Record Details">

```
Type: TXT
Name: _databuddy
Value: databuddy_abc123def456... (your unique token)
TTL: 300
```

</Tab>
<Tab value="DNS Format">

For the root domain `example.com`:
```
_databuddy.example.com TXT "databuddy_abc123def456..."
```

For subdomain `app.example.com`:
```
_databuddy.app.example.com TXT "databuddy_abc123def456..."
```

</Tab>
</Tabs>

</Step>
<Step>

## Verify Your Domain

1. **Wait for DNS propagation** (15-30 minutes typical)
2. **Return to your Databuddy dashboard**
3. **Click "Verify"** next to your domain
4. **Check status** - should change from "Pending" to "Verified"

<Callout type="success">
Once verified, you can create projects and start tracking analytics for this domain!
</Callout>

</Step>
</Steps>

## üåê DNS Provider Instructions

Choose your DNS provider for specific setup instructions:

<Accordions>
<Accordion title="Cloudflare">

**Step-by-step setup:**

1. **Log in** to [Cloudflare Dashboard](https://dash.cloudflare.com)
2. **Select your domain** from the list
3. **Go to DNS** ‚Üí **Records**
4. **Click "Add record"**
5. **Configure the record:**
   - **Type**: `TXT`
   - **Name**: `_databuddy`
   - **Content**: Your verification token
   - **Proxy status**: üîò DNS only (gray cloud) - **Important!**
   - **TTL**: Auto
6. **Save** the record

<Callout type="warn">
**Critical**: Make sure the proxy status is set to "DNS only" (gray cloud). Orange cloud will prevent verification.
</Callout>

**Verification time**: Usually 5-15 minutes

</Accordion>

<Accordion title="Namecheap">

**Step-by-step setup:**

1. **Log in** to [Namecheap Account](https://www.namecheap.com/myaccount/)
2. **Go to Domain List** ‚Üí **Manage** (next to your domain)
3. **Click "Advanced DNS"** tab
4. **Click "Add New Record"**
5. **Configure the record:**
   - **Type**: `TXT Record`
   - **Host**: `_databuddy`
   - **Value**: Your verification token
   - **TTL**: `Automatic`
6. **Save changes**

**Verification time**: Usually 10-30 minutes

</Accordion>

<Accordion title="GoDaddy">

**Step-by-step setup:**

1. **Log in** to [GoDaddy Account](https://account.godaddy.com/)
2. **Go to My Products** ‚Üí **DNS**
3. **Find your domain** and click **DNS** button
4. **Click "Add"** in the Records section
5. **Configure the record:**
   - **Type**: `TXT`
   - **Name**: `_databuddy`
   - **Value**: Your verification token
   - **TTL**: `1 Hour`
6. **Save** the record

**Verification time**: Usually 15-60 minutes

</Accordion>

<Accordion title="Route 53 (AWS)">

**Step-by-step setup:**

1. **Open** [Route 53 Console](https://console.aws.amazon.com/route53/)
2. **Go to Hosted zones**
3. **Select your domain**
4. **Click "Create record"**
5. **Configure the record:**
   - **Record name**: `_databuddy`
   - **Record type**: `TXT`
   - **Value**: Your verification token (with quotes)
   - **TTL**: `300`
6. **Click "Create records"**

**Verification time**: Usually 5-15 minutes

</Accordion>

<Accordion title="Google Domains">

**Step-by-step setup:**

1. **Log in** to [Google Domains](https://domains.google.com/)
2. **Select your domain**
3. **Go to DNS** tab
4. **Scroll to Custom records**
5. **Add a new record:**
   - **Host name**: `_databuddy`
   - **Type**: `TXT`
   - **TTL**: `300`
   - **Data**: Your verification token
6. **Save** the record

**Verification time**: Usually 10-30 minutes

</Accordion>

<Accordion title="Other Providers">

**General instructions for any DNS provider:**

1. **Access your DNS management panel**
2. **Add a new TXT record** with these settings:
   - **Name/Host**: `_databuddy`
   - **Type**: `TXT`
   - **Value/Content**: Your verification token
   - **TTL**: `300` or `5 minutes`
3. **Save the record**

**Common DNS interfaces:**
- Look for "DNS Records", "DNS Management", or "Zone File"
- Some providers use "Host" instead of "Name"
- Some providers use "Content" instead of "Value"

</Accordion>
</Accordions>

## üîß Troubleshooting

### Common Verification Issues

<Accordions>
<Accordion title='‚ùå "DNS record not found"'>

**Possible causes:**
- DNS record hasn't been added yet
- Incorrect record name (should be `_databuddy`)
- Wrong record type (should be `TXT`)
- DNS changes haven't propagated

**Solutions:**
1. ‚úÖ **Double-check record name**: Must be exactly `_databuddy`
2. ‚úÖ **Verify record type**: Must be `TXT` record
3. ‚úÖ **Wait for propagation**: Can take up to 48 hours
4. ‚úÖ **Test DNS**: Use [DNS Checker](https://dnschecker.org/) to verify

</Accordion>

<Accordion title='‚ùå "Verification token not found"'>

**Possible causes:**
- Token copied incorrectly
- Extra spaces or hidden characters
- Token expired or regenerated

**Solutions:**
1. ‚úÖ **Regenerate token**: Get a fresh verification token
2. ‚úÖ **Copy carefully**: Ensure no extra spaces or characters
3. ‚úÖ **Use quotes**: Some DNS providers require quotes around the value
4. ‚úÖ **Re-add record**: Delete old record and add new one with correct token

</Accordion>

<Accordion title='‚ùå "Record exists but verification fails"'>

**Possible causes:**
- Cloudflare proxy enabled (orange cloud)
- Multiple conflicting TXT records
- DNS caching issues
- Record not fully propagated

**Solutions:**
1. ‚úÖ **Disable Cloudflare proxy**: Set to "DNS only" (gray cloud)
2. ‚úÖ **Remove duplicates**: Delete any duplicate TXT records
3. ‚úÖ **Clear cache**: Wait 24 hours or flush DNS cache
4. ‚úÖ **Check globally**: Use DNS propagation checker tools

</Accordion>

<Accordion title='‚ùå "Domain format invalid"'>

**Possible causes:**
- Including `http://` or `https://`
- Including paths like `/path`
- Including ports like `:8080`
- Using IP addresses instead of domains

**Solutions:**
1. ‚úÖ **Use domain only**: `example.com` not `https://example.com`
2. ‚úÖ **No paths**: `example.com` not `example.com/app`
3. ‚úÖ **No ports**: `example.com` not `example.com:3000`
4. ‚úÖ **Use domains**: Not IP addresses like `192.168.1.1`

</Accordion>
</Accordions>

## üìö Advanced Configuration

### Multiple Domains

You can verify multiple domains for a single project:

<Callout type="info">
**Use cases for multiple domains:**
- Different environments: `staging.example.com`, `app.example.com`
- Multiple brands under one organization
- International domains: `example.com`, `example.co.uk`
</Callout>

**Each domain needs its own verification:**
1. Add each domain separately
2. Get unique verification token for each
3. Configure DNS record for each domain
4. Verify each domain individually

### Subdomain Considerations

**Root domain vs subdomain verification:**

- **Root domain** (`example.com`): Covers `www.example.com` automatically
- **Subdomain** (`app.example.com`): Only covers that specific subdomain
- **Wildcard**: Not currently supported

<Callout type="warn">
**Important**: If you use subdomains, verify each one separately. Verifying `example.com` does NOT automatically verify `app.example.com`.
</Callout>
